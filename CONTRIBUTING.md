# Contributing to TouchGuard

Thank you for your interest in contributing to TouchGuard! We welcome contributions from the community to help improve this project.

## How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally.
3.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/my-new-feature
    ```
4.  **Make your changes**. Ensure your code follows the existing style (4-space indentation).
5.  **Test your changes**:
    - Run `make test` to run the automated test suite.
    - Manually verify functionality (see `README.md` for testing instructions).
6.  **Commit your changes** with a clear and descriptive commit message.
7.  **Push to your fork**:
    ```bash
    git push origin feature/my-new-feature
    ```
8.  **Open a Pull Request** against the `main` branch of the original repository.

## Development Guidelines

-   **Code Style**: We use standard C style with 4-space indentation. Please keep the code clean and readable.
-   **Documentation**: Update `README.md` and `CLAUDE.md` if you change functionality or add new features.
-   **Testing**: Add or update tests in `tests/` if applicable.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub. Provide as much detail as possible, including:
-   macOS version
-   TouchGuard version
-   Steps to reproduce the issue
